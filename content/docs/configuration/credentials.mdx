---
title: Credentials File
---

`~/.iac-toolbox/credentials` — INI-format file. Mode `600`. Never committed to git (`.gitignore` is auto-created in the directory).

## Format

```ini
[default]
cloudflare_api_token = <token>
grafana_admin_password = <password>
docker_hub_token = <token>
docker_hub_username = <username>

[staging]
cloudflare_api_token = <staging-token>
```

## Behavior

- Each section name (e.g. `[default]`, `[staging]`) is a profile.
- The `--profile` flag on any command selects which section to read from.
- Missing keys in a named profile fall back to the `[default]` section.
- The file is created with mode `600` by `iac-toolbox init` or `iac-toolbox credentials set`.

## Supported keys

| Key | Used by | Notes |
|---|---|---|
| `cloudflare_api_token` | Cloudflare Tunnel | API token with Tunnel:Edit + DNS:Edit scope |
| `grafana_admin_password` | Grafana admin login | Admin password for Grafana UI |
| `docker_hub_token` | GitHub Build Workflow | Docker Hub personal access token |
| `docker_hub_username` | GitHub Build Workflow | Docker Hub username (optional, can be stored here) |
| `github_pat` | GitHub Build Workflow | GitHub personal access token |
| `cloudflare_tunnel_token` | Cloudflare Tunnel | Tunnel token (alternative to API mode) |
| `vault_token` | HashiCorp Vault | Vault authentication token |
| `grafana_api_key` | Grafana API | API key for programmatic access |
| `pagerduty_key` | PagerDuty integration | PagerDuty integration key (coming soon) |
